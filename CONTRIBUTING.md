# Contributing to jcsnotfunny

Thank you for your interest in contributing to this project! Below are guidelines to help you get started.

## How to File Issues

### Issue Templates

We provide issue templates to streamline the process of reporting bugs, requesting features, and submitting episode requests. Please use the appropriate template when creating an issue:

- **Bug Report**: Use the `bug_report.md` template to report any bugs or issues you encounter.
- **Feature Request**: Use the `feature_request.md` template to suggest new features or improvements.
- **Episode Request**: Use the `episode_request.md` template to request a new episode or guest.

### Label Usage

Labels help categorize and prioritize issues. Here's a quick guide to the labels we use:

- **Area Labels**: Indicate the area of the project the issue relates to:

  - `area/editing`: Video/audio editing tasks
  - `area/website`: Website and SEO tasks
  - `area/booking`: Booking and CRM tasks

- **Priority Labels**: Indicate the priority of the issue:

  - `priority/high`: High priority work
  - `priority/low`: Low priority work

- **Type Labels**: Indicate the type of work:

  - `type/automation`: Automation, scripts, or agents
  - `type/ops`: Operational tasks

- **Status Labels**: Indicate the status of the issue:

  - `status/blocked`: Blocked work
  - `needs/review`: Needs review or QA

- **Epic Labels**: Used for large initiatives or epics:
  - `epic`: Epic or large initiative

### Creating an Issue

1. Navigate to the [Issues](https://github.com/cbwinslow/jcsnotfunny/issues) tab in the repository.
2. Click on "New Issue".
3. Select the appropriate template for your issue.
4. Fill out the template with as much detail as possible.
5. Apply the relevant labels to your issue.
6. Click "Submit new issue".

## How to Contribute Code

### Fork the Repository

1. Fork the repository to your GitHub account.
2. Clone the forked repository to your local machine.

### Create a Branch

1. Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```

### Make Changes

1. Make your changes to the codebase.
2. Ensure your code follows the project's coding standards and best practices.

### Commit Changes

1. Commit your changes with a descriptive commit message:
   ```bash
   git commit -m "Add feature: your feature description"
   ```

### Push Changes

1. Push your changes to your forked repository:
   ```bash
   git push origin feature/your-feature-name
   ```

### Create a Pull Request

1. Navigate to the [Pull Requests](https://github.com/cbwinslow/jcsnotfunny/pulls) tab in the original repository.
2. Click on "New Pull Request".
3. Select your fork and the branch you created.
4. Provide a detailed description of your changes.
5. Click "Create Pull Request".

## Code of Conduct

Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.

## Questions or Need Help?

If you have any questions or need help, feel free to open an issue or contact the project maintainers.

Thank you for contributing to jcsnotfunny!
